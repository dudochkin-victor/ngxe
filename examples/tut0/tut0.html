Documentation

Every programmer knows that you go now where without documentation. A reference on the subject at hand is always important and very handy. I don't think very many of us learn by heart every detail of say the complete opengl specification.

We will assume we have atleast OpenGL 1.4. Yes Version 2.0 is out there. But most of the things in version 2.0 is already in 1.4 or 1.5 as extensions. Where differences occur I will point them out. We will come to the Shader language after a few tutorials. But there is a big number of things to cover before we ge there.

I suggest getting your hands on the OpenGL spec atleast version 1.4. Version 1.1 will probably do for the first few tutorials. As most of the real basics never changed. For a start documentation on the new shader language is not needed. I will specify documentation on that later. I have installed on my system man pages for all the opengl functions. But any sort of reference will come in very handy.

I will use the SDL library for basic setup of the screen and input from mouse/keyboard. SDL language is available for free on there web-site. The SDL API is fairly simple and you wouldn't really need the documentation for the tutorials. I will explain some of the basics as we go along.

Lastly I will use c++ as my programming language. Using the g++ compiler for now. No I'm not intrested in porting anything to visual c++ for now. There is many many documenation and books out there for learning c++. And I will pressume you know atleast the basics for this. I believe that OOP is the future. And in my opinion Java is not a game programming platvorm, that leaving c++ the only option for now. The D programming language developed by digital mars looks good and maybe I will look into it before the end of these tutorials. The idea of the tutorials is not to be bound to any specific language, but we need a language to program hey :)

I'm using GNU Debian Linux as a OS. I have a Geforce FX 5200 graphics card at the moment. That was infact the cheapest DirectX 9 card I could buy. I think my Nvidia drivers is out-dated by a few months. But so is just about every thing :(

What now

That is about it for tutorial 0. In tutorial 1 I will start with the basics of setting up a window and displaying something in it maybe. I will try to move beyond the basic immediate mode stuff as soon as possible and get to something feasable.

In later tutorials I will start working with models as such. The model program I recommend is Blender. Blender is competely free. And it supports Python plugins to export custom data. Python is not all that difficult and standard documentation is available with tutorials. Blender is in my opinion a most excelent model editor. Althought learning Blender is slightly more difficult than your average paint program. Many tutorials and documentation is available on the web. And I think it will be worth your while learning atleas the basics of blender as a feature game programmer. Atleast for the moment before you have somebody else that models things for you.
